## Step 3: Configure Environment Variables

### Create .env.local File

<CodeBlock language="powershell" code={`# On Windows PowerShell
New-Item -Path .env.local -ItemType File

# Or manually create the file in your project root`} />

### Add Your Configuration

<CodeBlock language="bash" code={`# ============================================
# Required: Gemini API Configuration
# ============================================
GEMINI_API_KEY=your_gemini_api_key_here

# ============================================
# Application Configuration
# ============================================
NEXT_PUBLIC_APP_URL=http://localhost:3000
NODE_ENV=development

# ============================================
# Optional: Production Configuration
# ============================================
# Firebolt (for production - uses mock in dev)
# FIREBOLT_CLIENT_ID=your_firebolt_client_id
# FIREBOLT_CLIENT_SECRET=your_firebolt_client_secret
# FIREBOLT_ACCOUNT=your_account_name
# FIREBOLT_DATABASE=your_database_name

# Gmail (for production - uses sandbox in dev)
# GMAIL_CLIENT_ID=your_gmail_oauth_client_id
# GMAIL_CLIENT_SECRET=your_gmail_oauth_client_secret
# GMAIL_REFRESH_TOKEN=your_refresh_token`} />

### Important Notes

- Replace `your_gemini_api_key_here` with your actual API key from [Gemini Setup](/firebolt?section=gemini-setup)
- Development mode uses mock Firebolt data and sandbox Gmail (no credentials needed)
- Production credentials are optional for this workshop

## Troubleshooting

### Port 3000 Already in Use

<CodeBlock language="bash" code={`# Kill process on port 3000
npx kill-port 3000

# Or use a different port
npm run dev -- -p 3001`} />

### Environment Variables Not Loading

- Ensure `.env.local` is in the project root (not in `src/`)
- Restart the dev server after adding/changing env vars
- Check for syntax errors (no spaces around `=`)
- Verify the file is named `.env.local` (not `.env.local.txt`)

### Gemini API Errors

- Verify your API key is correct (no extra spaces)
- Check you're not hitting rate limits (15 RPM on free tier)
- Ensure `GEMINI_API_KEY` is exactly as shown
- See [Gemini Setup Troubleshooting](/firebolt?section=gemini-setup#troubleshooting)

### Dependencies Failed to Install

<CodeBlock language="bash" code={`# Clear npm cache
npm cache clean --force

# Delete node_modules and reinstall
rm -rf node_modules package-lock.json
npm install`} />

## Workshop Modes

### Development Mode (Default)

- **Firebolt:** Uses mock data, no credentials needed
- **Gmail:** Sandbox mode, logs emails to console
- **Perfect for:** Learning, testing, local development

### Production Mode (Advanced)

- **Firebolt:** Connects to real Firebolt database
- **Gmail:** Sends actual emails via OAuth2
- **Requires:** Full setup of Firebolt and Gmail credentials

To enable production mode, set `NODE_ENV=production` and configure all credentials in `.env.local`.
